# Вынос базы знаний в отдельный Obsidian-vault + синхронизация

**Дата:** 2026-07-05 · **Статус:** дизайн одобрен, ожидает реализации

## Проблема
`docs/zazemli_knowledge_base/` (270 файлов, 13 МБ) вкоммичена прямо в репо сайта — это устаревший снимок БЗ **до** Obsidian-миграции. Актуальный канон переехал в отдельный приватный репозиторий `VesnaPenguin/zazemli-vault` (Obsidian-vault, активная рабочая память проекта с автокоммитами). Дублирование раздувает репо сайта и подсовывает агенту устаревшее.

## Находки разведки
- **vault = канон.** Миграция завершена (Вариант C — пофайлово новейшая версия по mtime; батчи 1+2+трек B). Вердикт Насти в `MIGRATION-STATUS.md`: «контентных пробелов миграции не осталось».
- **162 файла** старой БЗ отсутствуют в vault по basename, но объяснимо:
  - `.md`: в основном переименования — убраны суффиксы `_FULL`/`_vN`/`-7-SKU`, контент в vault под чистыми именами (`care-table`, `karta`, `market-research`, `договор-Ozon`, голос в `Бренд/Голос/*`). Дизайн-библиотека UI (~35 файлов) — намеренно ушла в Figma («дизайн через макет — канон»).
  - `bin`: шрифты живут в `docs/zazemli_design_arts/fonts/` (репо сайта, остаются); печать PDF партии 0 / QR / PNG-референсы — продакшн-ассеты в Google Drive.
- **Реально уникально** в старой БЗ (нет нигде в vault): 7 verbatim-досье (сырьё; обработанные досье в vault есть) + печатные PDF + PNG-референсы. Для разработки сайта не нужны.
- **Доступ к github из среды агента работает** через локальный прокси `127.0.0.1:8888` + git credential helper `store`. Клон и `pull` возможны (прошлый «висёж» лечится явным `http.proxy`).

## Решение
1. **vault снаружи репо сайта** — клон в `../zazemli-vault` (уже сделан). Репо сайта БЗ больше не содержит.
2. **Удалить `docs/zazemli_knowledge_base/`** через `git rm -r`. Историю не переписываем — уникальное сырьё восстановимо из git-истории + Drive.
3. **Обновить ссылки** в `AGENTS.md`, `DEVELOPMENT.md`, `my_tasks/*`: точка входа БЗ теперь `../zazemli-vault/НАВИГАЦИЯ.md` (внешняя, вне репо, «только чтение»).
4. **Синк-скрипт** `scripts/sync-vault.sh`: `git pull` vault через прокси (клонирует, если vault нет). vault двигает фаундер (Obsidian Git plugin, автокоммиты); агенту/разработчику — только pull перед работой.
5. **Не трогаем:** `docs/zazemli_design_arts/` (шрифты/дудлы/QR в коде), `docs/superpowers/` (наши спеки).

## Отвергнутые альтернативы
- **git submodule / subtree** — оставили бы БЗ формально внутри репо; выбран внешний клон.
- **Автосинк по cron/hook** — избыточно: vault обновляет фаундер нерегулярно, `pull` перед работой достаточно.
- **git filter-repo (переписать историю)** — инвазивно (force-push, смена всех хешей) ради экономии .git на pet-проекте не оправдано.

## Синхронизация
- Направление: **vault → локальный клон** (только pull). Правки БЗ делает фаундер в Obsidian, не агент.
- `scripts/sync-vault.sh` берёт прокси из `$HTTPS_PROXY`, ставит `GIT_TERMINAL_PROMPT=0`, опирается на credential helper `store`.
- Клон-путь по умолчанию `../zazemli-vault` рядом с репо сайта.

## Критерии готовности (verify)
- `npm run build` зелёный — БЗ не участвует в сборке (контент сайта в `src/content/`).
- `grep -rn zazemli_knowledge_base` по репо не находит живых ссылок (кроме архива/историч. артефактов).
- `scripts/sync-vault.sh` успешно делает pull `../zazemli-vault`.
