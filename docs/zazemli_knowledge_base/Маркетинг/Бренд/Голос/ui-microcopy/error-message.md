## Что это

UI-микрокопи для error-states на сайте `zazemli.com` в его текущем виде:

1. **404 not found**
2. **500 server error**
3. **Валидация формы подписки** (email)
4. **Network error** (нет связи)
5. **Maintenance / технические работы**

Это **не** human-кризисный ответ на отзыв или жалобу (для этого — `crisis-playbook.md`). Это микро-копи в самом интерфейсе, часто без человека на той стороне.

При массовом сбое (когда клиенты пишут в Direct «у вас сайт лежит») — error UI остаётся коротким, а в Direct отвечаем по `crisis-playbook.md`.

**Триггеры, которых НЕТ в этом артефакте (и почему):**

| Удалённый триггер | Почему |
|---|---|
| Empty state корзины | На сайте нет корзины — продажа через Ozon |
| Empty state «нет выбранных боксов» | Нет «избранного» / «подборки» в текущем сайте |
| Empty state дневников | Нет личного кабинета и нет дневников на сайте (дневник физический — в боксе) |
| Empty state заказов | Заказы — в кабинете Ozon, не у нас |
| Поиск без результатов | На сайте нет поиска — только лендинг с кнопкой Ozon |
| «Товар недоступен» | На сайте нет каталога — все боксы в Ozon |

Если в Спринте 8+ появится ЛК с дневниками/избранным — расширить артефакт обратно. Заготовки 3 удалённых сценариев сохранены в `_restored/HANDOFF_kopiraiting-task_2026-05-17-v1.md` (черновики draft v1.0).

## Роль в ДНК + канал обучения

- **Канал:** UI веба `zazemli.com`. Не сайт-копи (длинные блоки), не реклама, не email.
- **ДНК-доля:** Фармацевт 70% (ясность, факт, действие) + Друг 20% (теплота через ровный тон, не сюсюкание) + Натуралист 10% (рамки бренда — «бокс», «коллекция»).
- **Тон обучения:** НЕ обучаем. Error UI — это не место для манифеста, рецептуры или метафор «руки в землю». Только: что → почему → как.

## Формула (адаптация `crisis-playbook.md` под микро)

1. **Назвать ровно что произошло** — 1 короткая фраза, без вины пользователя.
2. **Дать факт-пруф** — причина одной строкой (без технических кодов в основном тексте; код можно мелким снизу).
3. **Дать 1-2 действия** — кнопка-глагол ≤15 знаков.
4. **(Опционально) Жест** — для крупных сбоев подпись «— Заземли». В мелких валидациях — не нужен.

---

## Слоты микроформата

| Слот | Длина | Правило |
|---|---|---|
| Статус | 1 слово или ≤4 слов | «Не нашлось» / «Сейчас не отвечаем» / «На паузе» |
| Причина | 1 фраза ≤80 знаков | нейтрально, без обвинения «вы ввели» |
| Действие (CTA) | глагол + ≤15 знаков | «Обновить», «На главную», «Попробовать снова», «На Ozon» |
| Подпись | опц., только для 500/network/maintenance | «— Заземли» |

## Шаблон + примеры (5 триггеров)

### 1. 404 not found

**Статус:** Страница не нашлась
**Причина:** Возможно, переехала или мы ошиблись со ссылкой.
**Действие:** На главную · Купить на Ozon

```
Страница не нашлась.
Возможно, переехала или мы ошиблись со ссылкой.

[На главную] [Купить на Ozon]
```

---

### 2. 500 server error

**Статус:** Сейчас не отвечаем
**Причина:** Наш сервер перегружен — уже смотрим.
**Действие:** Обновить
**Подпись:** — Заземли

```
Сейчас не отвечаем.
Наш сервер перегружен — уже смотрим.

[Обновить]

— Заземли
```

(Код `500` — мелким серым снизу страницы, не в основном тексте.)

---

### 3. Валидация формы подписки — некорректный email

**Микро, inline под полем:**

```
Похоже, в адресе ошибка. Проверьте ещё раз?
```

Без статуса-заголовка, без CTA — пользователь сам в поле.

**3a. Валидация — обязательное поле (если есть пометка required):**

```
Нужно заполнить.
```

---

### 4. Network error

**Статус:** Похоже, нет связи
**Причина:** Проверьте интернет и попробуйте снова.
**Действие:** Попробовать снова

```
Похоже, нет связи.
Проверьте интернет и попробуйте снова.

[Попробовать снова]
```

---

### 5. Maintenance / технические работы

**Статус:** На паузе до [время]
**Причина:** Идут плановые работы. Кнопка «Купить на Ozon» работает.
**Действие:** Купить на Ozon
**Подпись:** — Заземли

```
На паузе до [время].
Идут плановые работы. Кнопка «Купить на Ozon» работает.

[Купить на Ozon]

— Заземли
```

(В разработке: если maintenance занимает >2 часов — переключать `[Купить на Ozon]` на ozon.ru/seller/zazemli напрямую, чтобы не зависеть от нашего DNS.)

| Вариант | Текст | Когда |
|---|---|---|

## Правила применения

- **НЕ обвинять пользователя.** «Вы ввели неверный email» — нет → «Похоже, в адресе ошибка» — да. (anti-patterns §5.5)
- **НЕ извиняться слащаво.** «Простите-простите, очень виноваты» — нет → «Сейчас не отвечаем — уже смотрим» — да.
- **НЕ использовать «корзина», «оформите заказ», «закажите».** (word-list blacklist, anti-patterns §5.6) — у бренда нет корзины, продажа через Ozon.
- **Caveat = 0.** Микроформат не место для приписок. Исключение — вариант C для 404 (1 handwritten-нота, по правилу 11).
- **Эмодзи = 0.** Всегда, везде. (anti-patterns §5.3)
- **КАПС = 0** в потоке. (anti-patterns §5.2)
- **Восклицательные знаки = 0.** Точка по умолчанию.
- **CTA** — глагол, ≤15 знаков. Из whitelist: «На главную», «Обновить», «Попробовать снова», «Купить на Ozon».
- **«Купить на Ozon»** — главный CTA для 404 и maintenance, потому что сайт сейчас именно так монетизируется (см. шапку артефакта).
- **Domain в ссылках:** `zazemli.com` (откатано с `.space` 2026-05-17, решение A).
- **Подпись «— Заземли»** — только для крупных сбоев (500, network outage, maintenance). В валидации формы — нет.
- **Технический код ошибки** (500, 503, 404) — мелким второстепенным шрифтом снизу, не в основном тексте. Чтобы было видно при репорте бага, но не било в глаза пользователю.
- **Тон Наставник vs Подруга:**
  - Наставник — для технических (404, 500, network, maintenance).
  - Подруга — для пользовательских (валидация формы).
- **Расширение артефакта** — когда появится ЛК (Спринт 8+), вернуть удалённые сценарии (empty state корзины, дневников, поиск без результатов и т.д.). Их заготовки сохранены в `HANDOFF_kopiraiting-task_2026-05-17-v1.md`.

## Что НЕ применять

Точечно для error UI (см. полный список в `anti-patterns.md` §5):

| Запрещено | Почему |
|---|---|
| «Упс!», «Ой!», «Ой-ой!» | инфантилизм, сюсюкание |
| Эмодзи | TOV §1.3 — никогда нигде |
| «ОШИБКА!», «ВНИМАНИЕ!» в капсе | КАПС в потоке запрещён |
| «Простите-простите», «Нам очень-очень жаль» | сюсюкание; забота — через действие |
| «Internal Server Error 500 retry timeout exceeded» в основном тексте | технические термины пугают; коды — мелким снизу |
| «Попробуй ещё раз, может получится!» | неуверенный CTA + восклицание |
| «Вы ввели неверный email» / «Ваш запрос некорректен» | обвинение пользователя |
| «Добавить в корзину», «Оформите заказ» | у бренда нет корзины, продажа через Ozon |
| «Спасибо за терпение!» | продавцовское |
| «С теплом, команда Заземли» | сюсюкание; подпись — «— Заземли» |
| **Empty state корзины / дневников / заказов / поиска** | **На сайте сейчас этого нет — нерелевантно (решение C)** |

## Контр-пример (компактный, для тренировки)

```
Упс! Что-то пошло НЕ ТАК!
Простите-простите, наш сервер сошёл с ума и мы не знаем что делать.
Попробуйте обновить страницу, может повезёт!

С теплом и любовью, команда Заземли
```

**Нарушения (7 шт.):**
1. «Упс!» — инфантилизм
2. КАПС «НЕ ТАК!» в потоке
3. Восклицательные знаки ×3
4. «Простите-простите» — сюсюкание
5. «Сервер сошёл с ума» — нет конкретики, эмоциональная отсебятина
6. «Может повезёт!» — неуверенный CTA
7. «С теплом и любовью, команда» — запрещено TOV §3, anti-patterns §5.5

**Правильно:**

```
Сейчас не отвечаем.
Наш сервер перегружен — уже смотрим.

[Обновить]

— Заземли
```

## Чек-лист 10 anti-patterns (перед публикацией каждого микро)

1. Нет эмодзи?
2. Нет КАПСа в потоке (только функциональные коды снизу)?
3. Нет восклицательных знаков?
4. Нет слов из blacklist: «корзина», «оформите заказ», «приобретите», «уважаемый клиент», «спасибо за выбор», «премиальный»?
5. Нет обвинения пользователя («вы ввели», «вы сделали»)?
6. Нет слащавых извинений («простите-простите», «очень-очень жаль»)?
7. Нет «упс / ой / ого / йоу»?
8. CTA = глагол ≤15 знаков?
9. Caveat = 0 (исключение: 1 handwritten-нота на 404 вариант C, при условии что суммарно по коллекции ≤2; правило 11)?
10. Подпись «— Заземли» только для 500 / network / maintenance, не для валидации?
11. **Триггер из текущих 5 (404, 500, валидация email, network, maintenance)? Нерелевантные empty state не добавляем до появления ЛК.**

Если хотя бы один «нет» — переписать.
