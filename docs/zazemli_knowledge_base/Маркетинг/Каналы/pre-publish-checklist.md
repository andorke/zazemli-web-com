# pre-publish-checklist · Версия: 1.0.0 · Дата: 2026-05-17 · Источник: Спринт 10 (runbook'и), Настя 2026-05-17 (универсальный pre-publish чек-лист под все каналы)

> 📍 **Размещение:** `БЗ/Маркетинг/Каналы/pre-publish-checklist.md`. Применяется ко всем каналам — Ozon, IG, Telegram, email, сайт. Owner: CMO (Настя).

Чек-лист перед публикацией любого контента: TOV / бренд-кит / факт-чек / юр / UTM / коммерция / аналитика. Запускается **до** «Опубликовать», не после.

## Что это

Документ, который отвечает: **что проверить до того, как опубликовать пост / карточку / письмо / страницу, чтобы не пришлось удалять и переделывать.**

Не путать с:
- `Маркетинг/Бренд/Голос/anti-patterns.md` — справочник «как НЕ писать» (используется внутри чек-листа в шаге TOV).
- `Маркетинг/Бренд/Голос/crisis-playbook.md` — что делать **после** публикации, если случился кризис (см. также `Операционка/crisis-response.md`).
- `СЕО/brief-template.md` — это про **заказ** работы у подрядчика, pre-publish — про **финальную проверку** перед публикацией.

Краткая формула:
> **TOV → Бренд → Факт-чек → Юр → UTM/тех → Визуал → Коммерция → Аналитика → Публикация.**

## Когда использовать

✅ **Применяется к:**
- Постам и Reels в Instagram (`@zazemli_collectio`).
- Постам в Telegram (`@zazemli_collectio`).
- Карточкам Ozon (название + описание + bullet points).
- Email-рассылке (welcome-серия, дневник, промо).
- Лендингу на `zazemli.space` (любая публичная страница).
- QR-целевым страницам (`/lab`, `/guide`, `/collectio`, `/diary-signup`).
- Внешним материалам (PR-питч, статья, гостевой пост).

❌ **НЕ применяется к:**
- Внутренним документам (БЗ, рабочая папка).
- Прямой переписке в Direct/комментариях (там другой regime — см. `crisis-response.md` для отдельных кейсов).
- Сторис, исчезающим через 24 часа (легче чек: только §1 TOV + §3 Факт-чек).

## Универсальный чек-лист (8 категорий)

### 1. Голос (TOV)

- [ ] **Регистр соответствует каналу.** Ozon → Наставник, IG → Подруга, Сайт/Манифест → Редактор. См. `Маркетинг/Бренд/Голос/register-matrix.md`.
- [ ] **Нет запрещённых слов** «осознанность», «премиум-бокс». См. полный blacklist — `Маркетинг/Бренд/Голос/word-list.md`.
- [ ] **Нет анти-паттернов голоса** (хайп / клиширование / морализаторство). См. `Маркетинг/Бренд/Голос/anti-patterns.md`.
- [ ] **Шапка/первая строка** работает как hook (3-5 секунд для удержания).
- [ ] **Длина соответствует жанру** (Reels caption ≤ 220 знаков, IG post ≤ 1500, Ozon описание ≤ 6000, email subject ≤ 50).

### 2. Бренд (визуальная идентичность)

- [ ] **Палитра** — Bone/Charcoal/Moss + SKU-цвет (если применимо). Каноничная — `color-system.md`.
- [ ] **Шрифты** — Unbounded/Spectral italic/Caveat. Caveat ≤ 1 на печатный артефакт, ≤ 2 на web-коллекцию.
- [ ] **SKU AA-rule** (если печатное / мелкий текст): cosmos/iris/buttercup запрещены для текста <24px на bone — moss/charcoal вместо них.
- [ ] **Fleurons (❦)** — только moss, никогда SKU-цвет.
- [ ] **Логотип** — корректный variant (light/dark/mono), без искажений. См. `logo-system.md`.

### 3. Факт-чек

- [ ] **Все цифры** проверены против БЗ (Master Brief / channels / sku-system / legal-form / финмодель). Не выдумывать.
- [ ] **Все имена** (растений, рецептур, компонентов) — точные, без опечаток. Латынь сверена через `Продукт/Рецептуры/научное_обоснование`.
- [ ] **Все цены** соответствуют sku-system v1.0.2 (1 890 / 2 190 / 2 590 ₽). Sale/скидки — не используем в low season (см. channels §«Что НЕ применять»).
- [ ] **Все ссылки** проверены (открываются, ведут на нужную страницу, не 404).
- [ ] **Сезонные привязки** (даты, фестивали, акции) — корректные для года.

### 4. Юр. / комплаенс

- [ ] **Никаких обещаний результата** для конкретного покупателя («ваша монстера вырастет на 30%»). Только обобщённое из науки + caveat если нужен.
- [ ] **ИП-реквизиты** (для email-подвала, политики конфиденциальности, оферты) — актуальные.
- [ ] **Знак ™ или ®** на «ЗАЗЕМЛИ** — ТМ (заявка подана, регистрация ~12-15 мес после подачи). После регистрации — переключить на ®.
- [ ] **Согласие на ПДн** — для форм сбора email/telegram стоит чекбокс согласия + ссылка на политику конфиденциальности.
- [ ] **Реклама** — пометки «реклама» / ОРД ID — если посева в Telegram-каналах партнёров. Закон 38-ФЗ.
- [ ] **Маркировка возрастная** — не требуется для нашей категории (грунты).

### 5. UTM / технические параметры

- [ ] **UTM** проставлены по схеме из `channels.md` §9 + `Дизайн/UI/UX-стандарты/navigation-patterns.md`.
   - `utm_source` = канал (instagram / telegram / email / ozon / print).
   - `utm_medium` = формат (social / email / qr_box / qr_card / qr_journal).
   - `utm_campaign` = кампания (`launch_april`, `gift_december` и т.д.).
   - `utm_content` = креатив (`reels_asmr_monstera`).
- [ ] **Domain** — везде `zazemli.space` (не `zazemli.com`, не `zazemli.ru`).
- [ ] **Alt-теги** на изображениях — заполнены осмысленно (SEO + accessibility).
- [ ] **OG-теги** (для сайта) — title / description / image / type заполнены.
- [ ] **Размер изображения** — оптимизирован под канал (IG post 1080x1080, Reels 1080x1920, OG 1200x630, Ozon главное фото ≥ 700x700).

### 6. Визуал (фото/видео/графика)

- [ ] **Фото** — соответствует фото-брифу `Маркетинг/Каналы/Фото/photography.md` (natural light, рука-в-кадре, ботанический контекст).
- [ ] **Видео/Reels** — формат 9:16 для Reels, длительность ≤ 60 сек, есть субтитры (≥80% IG-аудитории смотрит без звука).
- [ ] **Дизайн графики** — соответствует UI-библиотеке `Дизайн/UI/Дизайн-библиотека/` или сделан в логике токенов.
- [ ] **QR-коды** — корректный canonical link + UTM, тест-скан сделан.
- [ ] **Никаких водяных знаков** стоков на фото.

### 7. Коммерция

- [ ] **CTA — одна** на единицу контента (не «купи + подпишись + поделись» в одном посте).
- [ ] **CTA-формулировка** соответствует тону канала. Ozon → «Купить» прямо. IG → мягкое («посмотреть на Ozon», «зайти в bio»). Email → одна основная кнопка.
- [ ] **Ссылка на покупку** — ведёт на правильный Ozon-SKU (не на главную, а на конкретный товар).
- [ ] **Промокод / скидка** — если используется, согласован с юнит-экономикой `legal-form.md` §4 (маржа не уходит в отрицательную зону).
- [ ] **Не давим продажи в low season** (окт-фев) — если в этот период есть пост → фокус на ретеншн/бренд, не на скидку. См. `channels.md` §«Что НЕ применять».

### 8. Аналитика

- [ ] **UTM** — см. §5 (двойная проверка).
- [ ] **События трекинга** — если есть скрипты аналитики на сайте, событие подвязано (signup_success, qr_scan, etc).
- [ ] **A/B-вариант** — если тестируем, есть второй вариант для контроля.
- [ ] **Метрика успеха** определена до публикации: что меряем (CTR / saves / conv / open rate / reply rate) — иначе пост = расход времени без learnings.

## Канал-специфика (что добавить к универсальному чек-листу)

### Ozon карточка

Дополнительно к §1-§8 — особое внимание на:
- **Регистр Наставник** (не Подруга, не Редактор) — плотно, по делу, никакого манифеста.
- **3-5 секунд на первый экран** — главное преимущество в первом bullet.
- **Характеристики** — все 16 SKU описаны единообразно (см. `Маркетинг/Каналы/Ozon/marketplace-card-ozon.md`).
- **Главное фото** — белый/нейтральный фон, без логотипа Ozon, без водяных знаков.
- **Запрещено сравнение** с HURY (даже косвенное) — `positioning.md` §3.
- **Отказное письмо** загружено в кабинет Seller.

### Instagram (пост / Reels / сторис)

- **Hashtags** — 5-12 штук, mix брендовых (#zazemli #COLLECTIO) и категорийных (#комнатныерастения #пересадка).
- **Геотег** — для пилота Москва-only.
- **Tag** партнёров если совместный контент.
- **Музыка Reels** — без авторских конфликтов (либо из библиотеки IG, либо собственная).
- **Сторис-стикер** опроса/вопроса — если применимо (повышает engagement).
- **Бренд-handle** в caption (`@zazemli_collectio`) — для discoverability.

### Email-рассылка

- **From-name** — «Настя из ЗАЗЕМЛИ» или «ЗАЗЕМЛИ» (тестировать что лучше работает).
- **Subject** ≤ 50 знаков, без CAPS, без избытка эмодзи.
- **Preview text** заполнен (первые 50-90 знаков письма).
- **CAN-SPAM эквивалент**: контакт ИП + unsubscribe-ссылка обязательны.
- **Test-send** на свой адрес + на адрес коллеги (проверить рендеринг в Gmail/Mail.ru/Apple Mail).

### Telegram канал

- **Превью ссылки** — отключить если ссылка длинная и портит вид (Instant View лучше всего работает для long-form контента).
- **Inline-кнопки** — если есть CTA в виде ссылки.
- **Закрепить** важный пост (welcome / актуальная акция).

### Сайт zazemli.space (лендинг, лаб, гайд)

- **Lighthouse score** ≥ 90 по Performance, ≥ 95 по Accessibility, ≥ 95 по SEO.
- **Мобильная адаптация** — проверить на iPhone и Android, ≤ 380px ширина.
- **404-страница** — бренд-голосом, не дефолт.
- **Sitemap.xml + robots.txt** — обновлены при добавлении новых страниц.
- **Schema.org** разметка (Product / Article / Organization) — для SEO.

## Quick-check (для микро-публикаций)

Если публикуешь сторис, ретвит, короткий комментарий — **3 вопроса** вместо полного чек-листа:

1. **Голос ок?** Нет запрещённых слов, регистр под канал.
2. **Факт ок?** Цифры, имена, ссылки верные.
3. **Бренд ок?** Если визуал — палитра/шрифты/SKU-rule соблюдены.

Если 3 ✅ → можно публиковать без полного цикла.

## Анти-паттерны

❌ **«Опубликую и потом проверю».** Через 30 минут после публикации удалить пост = потерять engagement и метрики. Митигация: чек-лист **до**.

❌ **«Чек-лист длинный, сделаю частично».** Каждый раздел нужен — пропуск §3 Факт даст ошибку, пропуск §4 Юр даст штраф или претензию. Митигация: для коротких форматов — quick-check.

❌ **«Подрядчик / агентство проверит за нас».** Подрядчик не знает бренд-кода ЗАЗЕМЛИ. Митигация: чек-лист — **наша** ответственность всегда, даже если контент готовит фрилансер.

❌ **«Голос подкорректирую правкой после публикации».** Алгоритмы IG/Telegram трактуют отредактированный пост как менее качественный → меньше показов. Митигация: финализировать **до**.

❌ **«UTM поставлю позже».** Потом не разобраться откуда трафик. Митигация: UTM на стадии черновика, не после.

## Правила применения

- **Запускается до публикации, не после.** Пройти чек-лист и зафиксировать дату/время прохождения.
- **Один чек-лист на одну единицу контента.** 5 постов = 5 проходов. Шорткаты («это похоже на прошлый, скип чек») = накопление ошибок.
- **Подрядчик не проверяет вместо тебя.** Чек-лист — внутренний контроль. Подрядчик отвечает за свой scope (см. `brief-template.md` §7).
- **Если найдена ошибка в чек-листе на этапе ревью** — не публиковать, исправить. Лучше задержка на час, чем удалённый пост.
- **Записывать learnings.** Если регулярно споткнулся на одном пункте → пересмотреть процесс выше (например, изменить шаблон контента).

## Что НЕ применять

- ❌ **Не использовать чек-лист как «галочки для галочек».** Цель — поймать ошибку, не пройти формальность. Если §3 Факт-чек прошёл за 10 секунд — почти наверняка не прошёл.
- ❌ **Не публиковать без §5 UTM** для всего, что ведёт на сайт. Аналитика без UTM = слепота.
- ❌ **Не запускать продажные акции в low season** через скидки. См. `channels.md` §«Что НЕ применять».
