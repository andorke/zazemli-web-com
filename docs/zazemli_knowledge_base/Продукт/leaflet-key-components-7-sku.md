## Что это

Synthesis из 7 back-листовок партии 0 + лицевой листовки в **единый источник правды** для копи и дизайна. На листовках партии 0 — выверенный финал ToV биотопов и рецептур. Используем дословно, не переписываем.

**Принцип листовок (важно для копи на сайте):**
- На back-листовке первая строка с компонентами — **главные 4 компонента грунта с %**, набраны самым большим шрифтом
- Остальные компоненты — второй строкой с %
- Под ними — текст-объяснение биотопа (3 предложения формула)
- Этот же принцип переносим в `/collectio` и /lab — фокус на **ключевые компоненты**, не на полную рецептуру

## Связь с core formula

**Core formula бренда:** «Земля и забота — всё, что нужно»

В контексте коллекции:
- «**Землю** мы дали (вот ключевые компоненты под твоё растение)»
- «**Заботу** — ты»
- «Дневник — мост»
- «Твоё растение скажет тебе спасибо»

Каждый бокс — это «земля» под конкретный род. Биотоп объясняет, **почему именно такая земля**:
- Антуриум → кора и кокос → потому что растёт на деревьях
- Замиокулькас → пеностекло и песок → потому что суккулент

Это **рациональный proof для эмоциональной формулы**. Земля — это не «грунт» в магазине, это **рецептура под биотоп**.

## Правила применения

- **Биотоп-формула дословно** — не переписываем (вышла после CPO-ресёча + ToV-выверки Насты, согласована с печатью партии 0).
- **Ключевые компоненты первые** — на любой витрине / в любой карточке. Не «11 компонентов» абстрактно, а «кора 20% + кокос 20% + мох 10% + цеолит 10%» конкретно.
- **Характерная фраза — Caveat-стиль** (handwritten-пасхалка из словаря, по `brand-voice-profile.md` v1.0.5 §Caveat-лимит, правило 11).
- **% видны везде, где есть место** — apothecary-точность.
- **Латынь только род** в Spectral italic (по `sku-system.md` §1 правила формата).
- **Слово «земля»** (не «грунт») — в манифестных текстах + core formula. **«Грунт»** — в технических / Ozon / лаб-контексте.

---

## Что НЕ применять

- ❌ **Не переписывать биотоп-формулы** — они выверены и напечатаны. Изменения только после нового CPO-ресёча + согласования Насты.
- ❌ **Не использовать только название компонентов** без % — нарушает apothecary-точность.
- ❌ **Не использовать латынь с видом** (deliciosa, elastica и т.д.) — только род (по правилу `ficus_universal`).
- ❌ **Не дублировать характерные фразы между SKU** — каждая уникальна, не переписываем.
- ❌ **Не использовать «как Aesop / Diptyque»** в публичной коммуникации (TOV-inspiration внутренний).
