# Требования к ресечу · карта исследований SKU N°001–007

**Назначение:** внутренний стандарт качества. По этому файлу Оркестратор собирает фактуру для досье и проверяет себя. CDO/COO/CFO на воротах сверяются с ним же.
**Версия:** 1.0 (утверждена Настей 2026-05-17 фразой «собирай фактуру»).

## 0. Скоуп тем (Q4=A · обновлён Настей 2026-05-17: добавлена тема №1 «Биотоп»)

Каждое из 7 досье покрывает **19 тем**. Перед ними — короткая «Карточка SKU» (канон по `sku-system.md`, латынь, SKU-цвет, размер).

**Тема №1 (фундамент, идёт первой во всех досье) — БИОТОП.** Из него вытекают все остальные требования. Подпункты:
- естественный ареал (страны, климат, высоты)
- ярус леса и тип жизни (нижний/средний/эпифит/гемиэпифит/лиана)
- освещённость в природе
- влажность в природе
- температура в природе
- субстрат в природе (подстилка, мох, кора)
- симбиозы и стратегии корней
- ключевые следствия для домашнего ухода — мостик к темам 2–7 и 11

**Базовые (а–е):**
- а) уход в домашних условиях средней полосы РФ (сводный компендиум)
- б) земля и компоненты субстрата (триплет-summary + ссылка на `Заземли_научное_обоснование_рецептур.md`)
- в) полив: частота, объём, метод, требования к воде
- г) уход зимой, включая досвет (длительность, спектр, лампы, мощность)
- д) увлажнение воздуха: целевая %RH, способы, риски
- е) потребность в свете: lx/PPFD, локация в квартире, ориентация окна

**Дополнительные (ж1–ж12):**
- ж1) пересадка: частота, признаки готовности, особенности корневой системы
- ж2) горшок: материал, объём, дренаж, шаг увеличения
- ж3) подкормки: график, NPK, формы (жидкие/палочки/гранулы), сезон
- ж4) вредители и болезни: профилактика, диагностика, безопасные меры
- ж5) температурный диапазон: оптимум, минимум, чувствительность к сквознякам
- ж6) симптомы → диагностика (желтеющий лист, бурые края, опадение, гниль)
- ж7) акклиматизация после покупки и транспортировки
- ж8) размножение (черенки, деление, отводки)
- ж9) токсичность для людей и животных
- ж10) скорость роста, ожидаемый размер, ключевые сорта внутри SKU
- ж11) гигиена листа, обрезка, протирание
- ж12) безопасность маркетинговых утверждений в РФ — какие фразы про «очищает воздух», «полезно для здоровья» можно/нельзя по закону о рекламе

**Объём задачи:** 7 SKU × 19 тем = **133 ячейки**, каждая в формате триплета (см. § 2). Биотоп — самая важная тема: служит обоснованием для всех остальных, на него ссылаются темы 2–7 и 11.

## 0.5. Verified-only режим (зафикс. Настей 2026-05-17, Q6=A)

**Каждый факт = верифицированный факт.** Это значит:

1. **URL реально открыт** (через WebFetch или равноценный инструмент), не сгенерирован по шаблону.
2. **Дословная цитата** из источника лежит в файле досье (одна-две фразы в кавычках или блоке `>` с английским оригиналом).
3. **Если на странице цитируемой цифры нет** — факт не записывается. Вместо него — пометка «В источнике [URL, дата обращения] цифра не найдена; продолжаем поиск».
4. **Дата обращения** обязательна (формат `2026-05-17`) — источники меняются, ссылка-якорь во времени.
5. **Запрещены формулировки** «по аналогии», «вероятно», «обычно так» — они допустимы только в блоке «гипотеза, требует проверки» и не считаются фактом.

**Соотношение трудозатрат:**
- Verified-факт: ~3–7 минут (поиск страницы, открытие, выписка цитаты).
- 1 SKU = ~30–60 минут на subagent при честной работе.
- 7 SKU × 30–60 мин = 4–7 часов реальной работы через несколько прогонок subagent.

**Контроль качества:** в каждом факте триплет дополняется блоком:

```
- **Цитата (verbatim):** «...exact phrase from source...»
- **URL:** https://...
- **Дата обращения:** 2026-05-17
```

Если этого блока нет — факт неверифицирован, в финал не идёт.

### 0.5.1 Параметры verified-пересборки (Q7=A, зафикс. Настей 2026-05-17)

**Качественные формулировки extension-сервисов цитируем как есть.** Если MOBOT/RHS/EDIS/Kew/NC State дают «bright indoor light, no strong direct sun» — пишем эту фразу с URL и датой. Не превращаем её в «10 000–20 000 lx», даже если эти цифры циркулируют в литературе.

**Цифры (lx, PPFD, %RH, NPK, °C-пороги, ml воды) — только из академических публикаций.** Поиск: WebSearch по конкретным терминам + проверка через Wikipedia с переходом на первоисточник + книги (Croat, Madison, Boyce). Если за разумное время (≤3 поисковых запроса на цифру) publication уровня S/A не найдена — пишем TODO: «Цифровой диапазон <параметра> для <вида> в источниках S/A не подтверждён за <число> запросов от <дата>; требует ручного поиска академической публикации».

**Запрещено в verified режиме:**
- Цифры из обобщающих блогов/порталов уровня B (GreenInfo, plantcaretoday, простые блоги садоводов).
- Метка «уверенность: подтверждено двумя источниками S/A», если фактически открыт только один URL.
- Привязка цифры к источнику S, если на странице этого источника цифра отсутствует.

**Допустимо:**
- Качественная формулировка S-уровня без цифры.
- Цифра с пометкой «уверенность: один источник A» + verbatim + URL.
- TODO «требует поиска академической публикации» для критичных параметров, по которым с разумным усилием цифру не нашли.

## 1. Иерархия источников (от сильного к слабому)

**Уровень S — опорные, можно цитировать без вторичной проверки:**
- Botanical gardens: Missouri Botanical Garden (MOBOT), Kew (POWO/Plants of the World Online), Royal Horticultural Society (RHS).
- Университетские extension-сервисы: UF/IFAS EDIS (Флорида), NC State Extension, Penn State Extension.
- Профильные общества: International Aroid Society (для антуриумов, монстеры, эпипремнума, спатифиллума, аглаонемы, замиокулькаса — 6 из 7 SKU — ароидные).
- Рецензируемые статьи в Scopus/Web of Science (PubMed для биологии — через bio-research:consensus).
- Уже верифицированный документ БЗ: `Продукт/Рецептуры/Заземли_научное_обоснование_рецептур.md` (80+ источников в нём уже проверены).

**Уровень A — допустимые, но желательно подтвердить вторым источником:**
- Книги ведущих растениеводов с ISBN и научным редактором (Croat, Madison по ароидным; русские — Семенова Н. GreenInfo).
- Производители субстратов и удобрений уровня Lechuza, Compo, Agrecol — для рекомендаций по концентрациям NPK и pH.
- Профильные англоязычные ресурсы с указанием авторов и источников: gardenia.net, plantcaretoday.com (только если ссылаются на extension-сервисы).

**Уровень B — справочный, только для проверки направления, не для прямых цифр:**
- Русскоязычные форумы и энциклопедии (GreenInfo, iplants.ru, flowers.bestolkovyj-narod.ru) — берём как индикатор «что обычно делают в РФ», но цифры всегда сверяем с уровнем S/A.

**Запрещено как первоисточник:**
- Блоги без указания автора и источников.
- Маркетплейс-описания (Ozon, WB) — это коммерческий копирайт, не факт.
- ChatGPT / Claude / Perplexity-ответы без проверки по первоисточникам — модель может галлюцинировать.
- Pinterest, Instagram-посты, TikTok — визуал нормально, фактура нет.

## 2. Что обязательно фиксировать на каждый факт

**Триплет факта (зафикс. Настей 2026-05-17):** каждое утверждение в досье оформляется тремя строками:

1. **ФАКТ.** Сухая формулировка с цифрой и единицей. Не «много света», а «10 000–20 000 lx» или «PPFD 50–150 μmol/m²/s» или «50–60 %RH».
2. **ЗНАЧЕНИЕ ДЛЯ НАС.** Что этот факт меняет в продукте/коммуникации/боксе. Пример: «значит, на северном окне в Москве монстера будет недополучать света 5–6 месяцев в году — нужен досвет; в карточке Ozon честно проговариваем».
3. **ПРИМЕНЕНИЕ В СРЕДНЕЙ ПОЛОСЕ РФ.** Конкретная инструкция для покупателя: где поставить, что купить, как часто, по какому признаку. Не «давайте достаточно света», а «окно south/east + 30–50 см от стекла; при north/west — фитолампа 14 Вт на 8–10 ч/сут с октября по март».

Метаданные к триплету:
- **Категория:** {освещённость, полив, влажность, температура, субстрат, досвет, пересадка, подкормка, токсичность, размножение, болезни, акклиматизация, прочее}.
- **Источник:** уровень (S/A/B) + название + ссылка + год.
- **Степень уверенности:** {подтверждено двумя источниками S/A} | {один источник S} | {один источник A} | {в источниках расхождение — см. «разногласия»}.

## 3. Правило «нет данных»

Если на тему по конкретному SKU цифры в источниках не нашлись или они противоречат друг другу настолько, что выводить медиану нечестно — в досье пишется дословно: **«В источниках не подтверждено по состоянию на 2026-05-17. Гипотеза по аналогии с [SKU/родом]: …, требует проверки.»** Никогда не выдумывать. Никогда не округлять в красивую цифру «на глаз».

## 4. Региональная привязка (фиксировано Настей)

Целевой регион — **Москва и средняя полоса РФ**. Это значит:
- Температура воздуха в квартире зимой при ЦО: 20–24°C, в отопительный сезон сухо (RH часто 20–35%).
- Длительность светового дня: 7 ч в декабре, 17 ч в июне (55°45′ с.ш.).
- Окна: ориентация и расстояние от окна — критичны для всех «light»-выводов. В досье отдельной строкой указывать «при окне на N/S/E/W».
- Качество воды: московская вода жёсткая (~3–4 мг-экв/л), это влияет на рекомендации по фильтрации / отстаиванию / pH.

## 5. Противоречия между источниками

Если RHS говорит «полив раз в неделю», а MOBOT «при подсыхании верхних 2 см» — фиксируем оба:
- Сначала более операциональная формулировка (по признаку, а не по календарю).
- В скобках — альтернатива и источник.
- В сноске «Расхождение источников: …» с пояснением, почему один надёжнее.

## 6. Гранулярность темы (что считается «закрытой»)

Тема закрыта, если в досье есть ответ на 5 вопросов:
1. Что делать (метод / частота / параметр).
2. По какому признаку определять, что пора (триггер).
3. Что бывает, если делать неправильно (типичные ошибки).
4. Как это применимо к домашним условиям средней полосы РФ.
5. Источник уровня S или два уровня A.

Если хоть один вопрос без ответа — тема в статусе «частично», в индекс-матрице маркируется жёлтым, в досье — TODO с конкретным подвопросом.

## 7. Юр.фильтр (важно — на воротах [COO])

Любые формулировки про «полезно для здоровья», «очищает воздух», «улучшает сон» — **не идут в карту без согласования COO**. Помечаются `[ПРОВЕРИТЬ COO]`. Это нужно потому, что закон о рекламе в РФ накладывает ограничения на медицинские/health-утверждения. Если в источнике (даже NASA Clean Air Study) есть такие выводы — пишем «по [источник]: воздействие на формальдегид/бензол выявлено в лабораторных условиях, перенос на бытовое применение спорный». Без оценочных эпитетов.

Тема «токсичность для людей и животных» (если войдёт по Q4) — обязательная, цитировать ASPCA, RHS Toxicity Database, обязательно «может вызывать раздражение слизистых при контакте/проглатывании», без «опасно» / «безопасно».

## 8. Лексика бренда

В тексте досье не используем:
- «осознанность», «премиум-бокс» — выкинуты из словаря.
- «полезно для здоровья», «улучшает самочувствие», «детокс воздуха» — юридический риск (см. §7).
- Декоративную латынь (`observatio`, `cyclus`). Латынь только для рода (Monstera deliciosa, Ficus elastica).

В тексте досье используем:
- Род и вид по канону: Monstera deliciosa, Ficus benjamina/elastica/lyrata (нужно уточнить, какой именно вид по каждому SKU), Anthurium andraeanum/scherzerianum, Aglaonema commutatum/modestum, Spathiphyllum wallisii/cochlearispathum, Zamioculcas zamiifolia, Epipremnum aureum.
- Бытовое имя — основное, латынь — в скобках при первом упоминании в досье.

## 11. Ворота качества перед сохранением в БЗ

Финал каждого досье — самопроверка по чек-листу:
1. Каждый числовой факт имеет источник уровня S или два уровня A.
2. Ни один факт не выдуман и не округлён «на глаз».
3. Нет запрещённой лексики (§8).
4. Нет health-утверждений без `[ПРОВЕРИТЬ COO]` (§7).
5. Региональная оговорка указана для каждого спорного параметра (§4).
6. Расхождения источников отражены, а не замазаны (§5).
7. Пересечение с `Заземли_научное_обоснование_рецептур.md` оформлено как ссылка, а не дубль (§9).

После самопроверки — ворота:
- **[COO]** — юр.риски, особенно токсичность и рекламные утверждения.
- **[CFO]** — фин.риски: если рекомендации тянут бюджет покупателя на досвет/увлажнитель/термометр, это влияет на позиционирование бокса (бокс — для пересадки, не для климатической установки квартиры).
- **[CDO]** — консистентность лексики и подачи, соответствие TOV v1.2.

## 12. Что НЕ входит в карту (явно)

- Контент-планы IG / тексты карточек Ozon. Карта — сырьё, не финальный копирайт.
- Размер визуала, фотобрифы. Это работа CDO, отдельные артефакты.
- Юнит-экономика. Это финмодель.
- Сравнение с HURY и другими конкурентами. Это работа CMO, отдельный артефакт.
