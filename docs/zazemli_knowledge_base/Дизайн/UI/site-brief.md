# site-brief — wrapper-документ

> **Что это.** Тонкий бриф для дизайнера сайта `zazemli.com`. Канон по визуалу/голосу/компонентам — в БЗ (карта в §3). Здесь — только бизнес-цель, конкретика по сайту и то, чего в БЗ нет.
>
> **Версия:** v0.5 · 2026-05-24 · Оркестратор. **Дедлайн MVP:** 2026-05-31. **Целевое размещение:** `БЗ/Дизайн/UI/site-brief.md`.

---

## 1. Главное

— Сайт = **витрина-музей**, не магазин. Покупка → внешне на Ozon (на старте — заглушки «Скоро · подпишись»).
— Сайт работает на 3 аудиториях: холодная (из IG/поиска), тёплая (QR с листовки), купившая (QR со стр III дневника → подписка на email-дневник).
— Принцип Насти: «максимально простой MVP — email-подписка + Ozon + лаборатория + гайд. Всё остальное → v2 / через 6 мес. Этот принцип сильнее любых других при конфликте».

### 4.5 Метрики — 7 точек измерения (вводные Насты, реализацию продумает Иван)

> **Дословно от Насты 2026-05-24:** «есть несколько мест, где мне нужно мерить — Иван чуть позже это продумает».

1. Переходы по QR-коду на `/collectio`
2. Переходы по QR-коду на `/guide`
3. Переходы по QR-коду на `/lab`
4. Переходы из `/lab` на `/collectio` кликом на странице сайта
5. Переходы из `/guide` на `/collectio` кликом на странице сайта
6. Проваливание из `/collectio` в карточку товара `/collectio/[plant]`
7. Клик «Купить» из карточки товара (переход на Ozon)

**Реализация — за Иваном.** Подсказки: для (1–3) — UTM-разметка в QR (`utm_source=qr&utm_medium=print&utm_campaign=partia-0`) + цели Я.Метрики. Для (4–5) — JS-эвент при клике на CTA или фильтр Я.Метрики по referrer. Для (6–7) — клик-эвенты Я.Метрики.
