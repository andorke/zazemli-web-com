# DS-Governance · Управление дизайн-системой ЗАЗЕМЛИ

**Версия:** 1.0 · **Дата:** 2026-05-26 · **Owner:** [Оркестратор]
**Контекст:** документ создан перед стартом ТРЕК 3 (Figma 27-29 мая) как защита от хаоса при расширении DS под рефы дип-ресёрча.
**Связь:** `brief-CDO.md` v1.1 (шорт-лист 12 типов блоков), `Дизайн/UI/Дизайн-библиотека/` (15 atoms + 6 molecules + 8 organisms + 7 templates), `color-system.md`, `typography.md`, `semantic-tokens.md`. Memory: `mvp_no_animation`, `typography_first_principle`, `sku_color_aa_rule`, `caveat_print_limit`, `fleurons_always_moss`, `epipremnum_moss_color`.

---

## Главный принцип

**DS = живая система. Она РАСШИРЯЕТСЯ под новые нужды, но НИКОГДА не НАРУШАЕТСЯ.**

Любой реф входит в DS как новый component (или вариант существующего), не как «исключение». Контракт на токены — священен.

«Шрифтов 3, не 10» — не лозунг, а **архитектурный закон**.

---

## §1 · Каскад приоритетов

Когда возникает конфликт (реф vs DS), решает каскад от высшего к низшему:

1. **Дизайн-принципы** — нельзя нарушить никогда:
   - No-animation (memory `mvp_no_animation`)
   - Typography-first, photo-as-element (memory `typography_first_principle`)
   - Genus-first навигация (memory `feedback_purchase_scenario_genus_first`)

2. **Бренд-токены** — нельзя нарушить:
   - `color-system.md` — палитра
   - `typography.md` — шрифты + размеры
   - `semantic-tokens.md` — отступы, радиусы, тени
   - AA-rule (memory `sku_color_aa_rule`)
   - Caveat-лимит (memory `caveat_print_limit`)
   - Fleurons всегда moss (memory `fleurons_always_moss`)

3. **Структура organisms из БЗ** — что есть в макете (какие блоки, в какой последовательности). Можно расширять через УВ.

4. **Параметры рефов** — как именно эти блоки выглядят (типографика, размеры, layout, hover). Из шорт-листа 12 блоков (`brief-CDO.md` v1.1).

5. **Креативное расширение CDO** — допустимо только в зоне «не покрыто 1-4». Эскалация Насте при сомнениях.

## §2 · Жёсткий контракт на токены

### Шрифты

Разрешено **только 3 family**:
- **Unbounded** — display (заголовки h1, h2, h3, display-1, display-2)
- **Spectral** — body (regular + italic для акцентов)
- **Caveat** — handwritten (только в Caveat-блоках, ≤2 на страницу)

**4-й шрифт = архитектурное решение** через УВ обновление `typography.md`. Не «по ходу макета», не «давайте просто возьмём Optima как у Aesop» или «Courier mono как у Le Labo». Если CDO в Figma испытывает потребность в новом шрифте → останавливается и эскалирует Насте.

### Цвета

Разрешено **только из** `color-system.md`:
- **База:** Bone (фон) / Charcoal (текст) / Moss (акцент)
- **SKU-цвета:** cosmos / iris / buttercup / moss / charcoal / poppy (6 уникальных значений)
- **Caveat-color** — приглушённый тёплый для handwritten-микрокопии

**SKU-распределение для 7 растений** (см. `color-system.md` + `brand-identity.md` v1.4 как актуальную карту):
- N°001 Монстера → **moss** (подтверждено Настей 2026-05-26)
- N°007 Эпипремнум → **moss** (подтверждено Настей 2026-05-26, memory `epipremnum_moss_color`)
- N°002-N°006 (Фикус, Антуриум, Аглаонема, Спатифиллум, Замиокулькас) → распределение по 5 оставшимся уникальным цветам (cosmos / iris / buttercup / charcoal / poppy) согласно `brand-identity.md`

Уникальность SKU-цветов нарушена сознательно — **moss закреплён за двумя SKU** (Монстера + Эпипремнум как родственные лианы тропиков).

**Новый цвет = архитектурное решение** через УВ `color-system.md`. Не silently добавлять «sepia как у Astier» или «терракот как у Cabana».

### Размеры шрифта

Только из typescale-токенов:
- display-1, display-2 (Hero)
- h1, h2, h3 (секции)
- body-lg, body, caption (тексты)

Кастомные font-size в макете = запрещены. Если нужен новый кегль — добавить в typescale через УВ.

### Отступы

Только из spacing-tokens (`semantic-tokens.md`): 4 / 8 / 16 / 24 / 32 / 48 px или xs / sm / md / lg / xl (в зависимости от текущей системы). Кастомные padding/margin = запрещены.

### Анимации

**Нет**. Только статика + CSS hover. Memory `mvp_no_animation`. Phase 1.5+ можно вернуться.

## §3 · 13-пунктовый Pre-commit чек-лист

**Перед добавлением любого нового atom/molecule/organism в DS — пройти 13 пунктов:**

- [ ] 1. Использует только шрифты из `typography.md` (3 family max)
- [ ] 2. Использует только цвета из `color-system.md` (без hardcoded `#`)
- [ ] 3. Использует только размеры из typescale-токенов
- [ ] 4. Использует только отступы из spacing-tokens
- [ ] 5. AA-rule прошло (контрастность ≥ 4.5:1 для small text, ≥ 3:1 для large text)
- [ ] 6. Caveat не превышен (на странице ≤2, на печатной ≤1)
- [ ] 7. Fleurons только moss (если есть)
- [ ] 8. Дудлы пропорциональные (width И height в одной пропорции)
- [ ] 9. В Figma привязан к Variables, не hardcoded
- [ ] 10. Имеет `.md` описание в `Дизайн/UI/Дизайн-библиотека/...`
- [ ] 11. Имеет уникальное название (нет дублей в БЗ — проверка через `check_bz_before_create`)
- [ ] 12. Есть варианты для mobile/desktop
- [ ] 13. Привязан к рефу из шорт-листа `brief-CDO.md` v1.1 ИЛИ к явному design-decision Насты (не «мне так захотелось»)

**Если хоть один пункт fail → не добавляем в DS, эскалация Насте.**

## §4 · 4 сценария расширения DS под реф

### Сценарий A · реф нуждается в новом элементе → создаём atom

**Пример:** Aesop Spotlight цитата под названием карточки SKU.
**Действие:**
1. Создать `.md` спецификацию в `Дизайн/UI/Дизайн-библиотека/Базовые-элементы/spotlight-quote.md` через УВ
2. В Figma создать Component `SpotlightQuote` с привязкой к Variables (Spectral italic, body, charcoal)
3. Использовать в существующем ProductCardMini как opt-in slot
4. Обновить НАВИГАЦИЯ.md (одна строка)

### Сценарий B · реф нуждается в составной структуре → создаём molecule

**Пример:** Aesop Ingredients accordion (photo + название + 1-2 строки × 11).
**Действие:**
1. Создать `.md` в `Дизайн/UI/Дизайн-библиотека/Составные-элементы/ingredient-stack.md` через УВ
2. В Figma — Component `IngredientStack` со slots: image / label / description
3. Использовать в `/lab` (MaterialsGrid 11 компонентов) или в InfoBlock на `/collectio`
4. Обновить НАВИГАЦИЯ.md

### Сценарий C · реф = новый вариант существующего organism

**Пример:** Hauser & Wirth Exhibitions = CollectionGrid + большие типографические section-разделители.
**Действие:**
1. Расширить `CollectionGrid` через УВ — добавить prop `withSectionDividers: boolean (optional)`
2. В Figma — добавить вариант существующего Component (Variant)
3. Existing CollectionGrid НЕ ломаем, дополняем

### Сценарий D · реф = новая секция → создаём organism

**Пример:** Curated Copenhagen «The Edit» — кураторские подборки.
**Действие:**
1. На MVP **не делаем** (Phase 1.5+)
2. Когда понадобится — создать `.md` в `Дизайн/UI/Дизайн-библиотека/Секции-страниц/edit-collection.md` через УВ
3. Зарезервировать имя и пометить в DS как «planned for Phase 1.5+»
4. Обновить НАВИГАЦИЯ.md

## §5 · Audit-протокол (еженедельно или после спринта)

**Через Figma Library Analytics** или плагин Style Lint:

1. **Hardcoded цвета в файле** — целевое значение **0**
2. **Hardcoded font-size** — целевое значение **0**
3. **Дублирующие компоненты** — слить (если случайно создали 2 разных EmailForm — выбрать один, заархивировать второй)
4. **Неиспользованные компоненты** (мёртвый код) — удалить или пометить `[archived]`
5. **«Toby-проверка»** — открыть случайные 3 макета и спросить: «всё ли тут из DS?». Если хоть один элемент «не из библиотеки» — флажок, разбор

**Owner аудита:** [Оркестратор] перед каждым релизом + [CDO] (виртуально через Настю) в ходе работы.

## §6 · 2 опасных кейса в шорт-листе рефов

**До старта Figma** обсудить с Настей (HTML-демо в работе, таск #10):

### Кейс 1 · Astier de Villatte sepia ink

**Реф:** D2.4 в коллаже — handwritten labels на product-карточках, sepia-чернила (не black ink), угол 3-5°.

**Риск:** sepia — потенциально новый цвет, которого может не быть в color-system.

**Решение:**
- Проверить: подходит ли existing `caveat-color` (уже определён в `color-system.md` как «приглушённый тёплый»)?
- Если подходит → использовать `caveat-color` для всех Caveat-блоков. **Не добавляем sepia как отдельный токен.**
- Если не подходит → расширение `color-system.md` через УВ + новый токен `caveat-sepia`. Архитектурное решение, эскалация Насте.

**Статус:** ОЖИДАЕТ HTML-демонстрации (таск #10) и решения Насты.

### Кейс 2 · Le Labo Courier mono для `/lab`

**Реф:** A2.2 в коллаже — Courier-like mono для всего, «typewriter-document» подача (manifest на /lab как реальный документ).

**Риск:** mono = 4-й шрифт. Нет в `typography.md` (только Unbounded, Spectral, Caveat).

**Решение:**
- Альтернатива А: использовать **Spectral regular** в `/lab` — нет mono-эффекта, но typography целая. Сохраняем 3 family.
- Альтернатива B: использовать **Unbounded Light в малом кегле + большой letter-spacing** как «технический» вид. Имитация документа без mono.
- Альтернатива C (требует решения): расширение `typography.md` через УВ + новый токен `mono-font` (IBM Plex Mono / JetBrains Mono / Geist Mono). Архитектурное решение, эскалация Насте.

**Статус:** ОЖИДАЕТ HTML-демонстрации (таск #10) и решения Насты.

## §7 · Эскалация — когда CDO останавливается

CDO в Figma останавливается и эскалирует Насте **в этих 6 ситуациях**:

1. Хочется добавить 4-й шрифт
2. Хочется добавить новый цвет
3. Реф противоречит дизайн-принципу (no-animation / typography-first / genus-first)
4. Реф противоречит бренд-токену
5. Не находится подходящий atom/molecule/organism в DS, и нужно создать новый — но критерии создания не очевидны
6. Реф нравится визуально, но при адаптации под наши токены теряется суть — обсудить, стоит ли брать реф вообще

**Не эскалировать** — если решение лежит в зоне 5 каскада (креативное расширение в зоне «не покрыто 1-4», все 13 чек-пунктов пройдены).
