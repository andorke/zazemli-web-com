# mockups · Версия: 1.0.0 · Дата: 2026-05-16 · Источник: brand-v3 §07 «Применения» + brand-v3 §13 «Мокапы страниц» + Спринты 4-6 mock-валидации

Pattern слоя `04_patterns/`. Каталог-индекс всех мокапов / application-примеров бренда: где живёт каждый пример, какие компоненты реализуют, каков статус (текущий / устаревший / запланирован).

## Что это

Live-каталог визуальных мокапов. brand-v3 §07 и §13 содержат примеры применений бренда (Hero DARK/LIGHT/TRIO, PRODUCT CARDS, WHISPER MODE, открытка-вкладка, мокапы 3 страниц). После декомпозиции в Спринтах 3-6 они остаются как gallery — этот pattern формализует, где какой мокап живёт, какой статус, чем заменён.

## Роль в ДНК-четвёрке + канал обучения

Метa-pattern — не несёт ДНК сам по себе, индексирует существующие мокапы. См. `02_voice/dna.md` §1, §7 для исходной ДНК.

## Правила применения этого каталога

- **При добавлении новой композиции в `04_patterns` / templates** — фиксировать здесь как новую строку.
- **При устаревании источника** (как brand-v3 §13 с «корзиной») — статус 🟡 + ссылка на правку в Спринте 11.
- **При создании нового mock-файла** — добавлять в «Каталог mock-файлов задачи».
- В v2+ этот pattern перерастает в **публичный mockup-каталог** (страница `/mockups/` или Figma library / Storybook).

## Статусы

| Иконка | Значение |
|---|---|
| ✅ | Актуален, реализован в DS. |
| 🟡 | Устарел в источнике, требует правки (Спринт 11). |
| 🔴 | Конфликт, заблокирован, требует решения. |
| 🆕 | Запланирован, кандидат на новый artifact. |

## Что НЕ делать

- Не создавать дубль application gallery как **новый файл** (brand-v3 §07 — каноничный источник, не переписывается).
- Не показывать устаревшие мокапы (13.A, 13.B, 13.C) как референс — только с пометкой «устарел».
- Не превращать mockup-каталог в Figma-library без отдельного решения (это другой инструмент).
