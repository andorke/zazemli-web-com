# RitualStep · Версия: 1.0.0 · Дата: 2026-05-14 · Источник: brand-v3 §15 (Материальная архитектура) + DS v1.0 §5.2 + §6.4 + dna.md §1, §4

Молекула слоя `03_components/molecules/`. **Универсальный шаг любого ритуала линейки заботы** — пересадки, подкормки, укоренения, реанимации. В MVP применяется только в ритуале пересадки (4 шага 00–03).

## Что это

Один шаг ритуала в формате карточки: цифра, заголовок, описание, рукописная подсказка. Используется как ячейка в `RitualSequence` (organism, Спринт 5). Универсальная архитектура: ритуал может состоять из любого числа шагов; нулевой шаг — опциональный pre-action / arrival.

## Роль в ДНК-четвёрке

RitualStep — мост **02 «Фармацевт» ↔ 03 «Натуралист»** (см. dna.md §4):
- Number + Title (Unbounded, аптекарская строгость, источник 02) — это апотекарный протокол описания шага.
- Description Spectral italic + Caveat-hint (источник 04 «Друг» через рукописную наивность) — личный голос внутри протокола.
- Привязка к физическому материалу (грунт, дренаж, перчатки — источник 03) — материальная фактура каждого шага.

Доля источников на носителе (по dna.md §7): ≈ 15% Галерист + 40% Фармацевт + 25% Натуралист + 20% Друг.

## Состав

| Слой | Что | Токен / атом |
|---|---|---|
| Контейнер | flex column, p-8, min-h-260px, фон условный (`isFirst` = moss, иначе charcoal) | semantic-tokens.md `--surface-deep` / `--accent-active` |
| Number | Unbounded 900, text-5xl (60 px), leading-none, цвет moss на charcoal-фоне ИЛИ bone/95 на moss-фоне | typography.md DisplayBlack |
| Title | Unbounded 900, text-base, mt-4, leading-tight, КАПС текст | typography.md Body-bold |
| Description | Spectral italic, text-sm, opacity 0.9, mt-auto pt-4, leading-relaxed | typography.md Narrative-italic |
| Hint | Caveat, text-lg, цвет buttercup, mt-2, префикс « → » | typography.md Caveat |

**Props (из DS §5.2):**

```jsx
<RitualStep
  number="00"
  title="ПРИХОД"
  description="Достань коробку, поставь на стол, выдохни. Ритуал начался."
  hint="можно не торопиться"
  isFirst={true}
/>
```

| Prop | Тип | Описание |
|---|---|---|
| `number` | string | Двузначная строка `"00"`, `"01"`, …, `"N"`. Не number — для сохранения leading-zero. |
| `title` | string | КАПС, до 4 слов. |
| `description` | string | 1–2 предложения, Spectral italic. |
| `hint` | string | До 5 слов, без точки, без эмодзи. |
| `isFirst` | boolean (default `false`) | Флаг для moss-фона нулевого / стартового шага. |

## Нумерация в MVP-ритуале пересадки

Опираясь на brand-v3 §15 «Материальная архитектура» (послойная сборка коробки):

| Number | Title | Описание (DS / brand-v3) | Уровень коробки | Hint |
|---|---|---|---|---|
| **00** | **ПРИХОД** | Льняной мешочек, аромат, «можно не торопиться» | уровень 06 (v2+) | можно не торопиться |
| **01** | **ЗАБОТА ДЛЯ КОРНЕЙ И РУК** | Перчатки + корневин | уровень 04 (чёрный конверт) | перчатки + корневин |
| **02** | **НАДЁЖНЫЙ ДРЕНАЖ** | Керамзит на дно горшка | уровень 02 (белый пакет без окошка) | белый пакет без окошка |
| **03** | **ВКУСНЫЙ ГРУНТ** | Грунт под конкретное растение | уровень 03 (белый пакет с окошком) | белый пакет с окошком |

> **MVP-уточнение:** «00 ПРИХОД» (льняной мешочек) — **v2 · со следующей партии** (brand-v3 §15). В партии 0 ритуал начинается с 01. Архитектурно `isFirst` ставится на 01 — moss-фон активирует первый actionable step.

## Линейка ритуалов (extensibility, dna.md §1.1.1)

RitualStep — **универсальный примитив**, не «пересадка-only». Архитектура поддерживает горизонт линейки ритуалов заботы:

| Ритуал | Когда | Шагов | MVP / v2+ |
|---|---|---|---|
| Пересадка | сезонно | 3 (01–03) или 4 (00–03 в v2+) | MVP |
| Подкормка | ежемесячно | 3 | v2+ |
| Укоренение | ad-hoc | 5 | v2+ |
| Реанимация | crisis | 6 | v3+ |
| Ритуалы для человека («Заземлись») | TBD | TBD | v3+ |

Каждый ритуал — массив `steps`, передаётся в `RitualSequence` (organism, Спринт 5).

## Правила применения

1. **Number — leading-zero двузначная строка.** «1» → `"01"`. Это сохраняет визуальный вес цифры.
2. **`isFirst` — ровно одна** в ритуале. Стартовый шаг получает moss-фон, остальные — charcoal.
3. **Title — макс 4 слова в КАПСе.** Длиннее — нечитаемо в Unbounded 900.
4. **Description — 1–2 предложения**, императив, Spectral italic. Не нумерованные списки, не маркетплейс-лексика.
5. **Hint — до 5 слов**, Caveat, всегда без эмодзи и без восклицания. Это рукописная маргиналия, не CTA.
6. **«00» — только если ритуал имеет pre-action / arrival.** Иначе ритуал начинается с «01». См. таблицу выше.

## Что НЕ применять

- Number как числа в формате `1` или `1.` (без leading-zero).
- Эмодзи в hint (нарушает источник 04 — рукописная наивность через шрифт, не через символы).
- Title в нижнем регистре (нарушает читаемость).
- Description с маркетплейс-лексикой («Добавьте в корзину», «Закажите сейчас») — см. anti-patterns.md §5.6.
- Более одного `isFirst={true}` — moss-фон перестаёт работать как акцент стартового шага.
- Использование RitualStep вне `RitualSequence` или без явного page-level KickerHeader-заголовка (нарушает архитектуру DS §6.4).
