# Требования к продукту

> Функциональные (FR) и нефункциональные (NFR) требования.

> **Пример заполнения:** `example_docs/REQUIREMENTS.md`

## Описание продукта
Маркетинговый сайт `zazemli.com` D2C-бренда ЗАЗЕМЛИ (боксы для пересадки комнатных растений, 7 SKU). Сайт — «витрина-музей» и бесплатные магниты доверия (лаборатория грунта, гайд по пересадке); сам не продаёт — единственная конверсия — внешние ссылки на Ozon. Тон: тихо, дорого, музейно, дружелюбное комьюнити.

## Бизнес-контекст
### Целевая аудитория
Три потока (источник: site-brief §1):
1. **Холодная** — из Instagram и поиска: знакомится с брендом, уходит на Ozon.
2. **Тёплая** — по QR с листовки в боксе: углубляется в `/lab` и `/guide`.
3. **Купившая** — по QR со страницы III печатного дневника: подписка на email-дневник (`/diary-signup`, retention).

### Критерий успеха MVP
- Сайт в продакшене: структура 5 страниц + собранная главная.
- 7 точек Яндекс.Метрики измеряют воронку QR → сайт → Ozon.
- Lighthouse-бюджет соблюдён (бриф CDO v1.2).

## Функциональные требования

### FR-1: Карта сайта MVP — 5 страниц
- **FR-1.1** `/` — главная (лендинг), эталон — Figma node 185:2.
- **FR-1.2** `/collectio` — витрина 7 SKU, некликабельная на MVP (без страниц `/collectio/[plant]`).
- **FR-1.3** `/lab` — лаборатория грунта (11 компонентов).
- **FR-1.4** `/guide` — гайд по пересадке (5 шагов).
- **FR-1.5** `/diary-signup` — подписка на email-дневник. **НЕ в меню и НЕ в футере** — вход только по QR из печатного дневника.
- **FR-1.6** На старте страницы FR-1.2–1.5 — заглушки; контент приходит от Насти по мере готовности Figma.

### FR-2: Навигация
- **FR-2.1** SiteHeader: wordmark ЗАЗЕМЛИ + меню «Коллекция · Лаборатория · Гайд» + кнопка «Купить на Ozon».
- **FR-2.2** SiteFooter: юр-инфо ИП (ОГРНИП/реквизиты), дисклеймер «Растения — не лекарство…», соцсети текстом (IG/TG `@zazemli_collectio`, без иконок), email `team@zazemli.com`, QR (без diary), гигантский wordmark ЗАЗЕМЛИ.
- **FR-2.3** Мобильная навигация — burger (микро-система UI из брифа CDO).

### FR-3: Конверсия на Ozon
- **FR-3.1** Кнопки «Купить на Ozon» — внешние ссылки с UTM `utm_source=site`.
- **FR-3.2** На `/collectio` — прямой Ozon-линк per-SKU (`utm_content=sku00X`); до готовности карточек Ozon — заглушки «Скоро на Ozon».

### FR-4: Юридический минимум (152-ФЗ)
- **FR-4.1** Чекбокс согласия на обработку ПДн + ссылка на политику конфиденциальности на форме `/diary-signup`.
- **FR-4.2** Cookie-баннер; по умолчанию необязательные cookie отклоняются.

### FR-5: Аналитика — Яндекс.Метрика, 7 точек
1. Переходы по QR на `/collectio`; 2. QR на `/guide`; 3. QR на `/lab`; 4. клик `/lab` → `/collectio`; 5. клик `/guide` → `/collectio`; 6. проваливание `/collectio` → карточка SKU (post-MVP); 7. клик «Купить» → Ozon.
- **FR-5.1** QR-переходы считаются по UTM (`utm_source=qr&utm_medium=print`), внутренние клики — JS-целями Метрики.

### FR-6: SEO-минимум
- **FR-6.1** title / description / canonical на каждой странице, sitemap, robots.
- **FR-6.2** OG-картинка универсальная. На `/guide` — Schema.org `HowTo`.

## Нефункциональные требования

### NFR-1: Производительность
| Метрика | Требование |
|---------|-----------|
| Lighthouse | Бюджет из брифа CDO v1.2 |
| Анимация | Только composited-свойства (transform/opacity), 0 тяжёлых либ |
| Изображения | Контейнеры с фиксированными соотношениями — без CLS |

### NFR-2: Надёжность
Статическая генерация (SSG) — сайт работает без рантайм-зависимостей от CMS/БД. Доступность из РФ обязательна (Vercel не рекомендован).

### NFR-3: Безопасность
Форма `/diary-signup`: валидация (react-hook-form + zod), согласие 152-ФЗ. Никаких прочих персональных данных сайт не собирает.

### NFR-4: Совместимость
`prefers-reduced-motion: reduce` глобально отключает анимацию. Контрасты — по матрице a11y из `tokens.json` (charcoal/bone = AAA).

### NFR-5: Поддерживаемость
Популярный стабильный стек (канон фаундера): Next.js + React + TypeScript strict + Tailwind + shadcn/ui. Дизайн-токены — единый источник (`tokens.json` → Tailwind-тема). Расширяемость: каталог (`/collectio/[plant]`), магазин и CMS подключаются позже без перестройки каркаса.

## Ограничения MVP (out of scope)
1. Корзина / онлайн-оплата (продажи только через Ozon, до v2 ~6 мес).
2. Страницы `/collectio/[plant]`, `/lab/[plant]`, `/guide/[plant]` — v2.
3. Калькулятор грунта — v2 после кастдева.
4. Email-форма на главной (lead-capture только `/diary-signup`).
5. Telegram-канал/бот на `/diary-signup` — только email.
6. Полный SEO (ключи, JSON-LD кроме HowTo, OG-генерация) — отдельное направление через ~6 мес.
7. Любая аналитика кроме Яндекс.Метрики (без GA/Hotjar/Vercel Analytics).
8. Блог, личный кабинет, мультиязычность, A/B-инфраструктура, чат-виджет, поиск по сайту.
9. Анимация — MVP без неё (принцип «MVP без анимации»); motion-токены заложены в DS на будущее.

| Термин | Определение |

| SKU N°001–N°007 | 7 растений коллекции; 16 товарных позиций = 7 растений × 2–3 объёма |

| Дневник | Печатный дневник ухода в боксе; его QR — единственный вход на `/diary-signup` |
| 60/30/10 | Цветовое правило: bone 60% / charcoal 30% / moss 10% |
| Core formula | «Земля и забота — всё, что нужно» — обязательна в Hero `/` и `/diary-signup` |
