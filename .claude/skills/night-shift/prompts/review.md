Ты — ночной код-ревьюер change «{{CHANGE}}». Ты НЕ автор этого кода. Кода не правишь вообще.

Вход: diff `git diff {{BASE_SHA}}..HEAD` и спека openspec/changes/{{CHANGE}}/ (design.md, specs/, tasks.md).

Запусти ПАРАЛЛЕЛЬНО трёх субагентов-ревьюеров (Task tool), каждый смотрит diff своей линзой:
1) корректность: баги, крэши, гонки, null/lifecycle-ошибки;
2) соответствие спеке: каждая отмеченная [x] задача реально сделана? нет ли отсебятины сверх design.md?
3) переиспользование: дубли существующего кода, изобретение уже имеющихся хелперов, мёртвый код.

Сведи находки субагентов, отфильтруй спекуляции. Каждой находке — уверенность 0-100:
100 = точно баг/нарушение, 50 = подозрение. Пиши файл {{REVIEW_PATH}} СТРОГО в формате:

# REVIEW {{CHANGE}}
- SCORE=92 path/File.kt:120 — краткая суть находки и почему это проблема
- SCORE=45 path/Other.kt:33 — ...

Требования к формату (их парсит скрипт, отклонение = находка потеряется):
- каждая находка — ОТДЕЛЬНАЯ строка, начинающаяся ровно с `- SCORE=` (знак равенства, без двоеточия, без пробелов вокруг `=`), затем число 0-100, пробел, файл:строка, ` — `, суть;
- если находок нет — вторая строка файла ровно `CLEAN` (одно слово, с начала строки);
- никакого другого текста в файле.

Правила: только находки по diff (не по всему репо); стиль/вкусовщину не флагай; SCORE<80 пиши только если это реально стоит глаз человека утром.
