# lab-page — spec (new)

## ADDED Requirements

### Requirement: Страница лаборатории по прототипу lab.html
`/lab` SHALL воспроизводить структуру пересобранного прототипа `lab.html` (пересборка 2026-07-05, inverted pyramid — главный ответ первым): hero («Лаборатория грунта.», sub «Земля, собранная под конкретное растение, а не одна на всех.») → проблема («Универсальный грунт не под твоё растение.», две колонки: прозаический поток + врезка-pullbox с научным якорем аэрации) → **7 рецептур N°01–N°07** («По одной земле на каждое растение.», карточки с инлайн-раскрытием, полоска долей g4, биотоп + состав с MaterialDot, суммы процентов = 100) → **11 компонентов** («Из чего собраны эти земли.», фото-грид из 11 плиток-кнопок, описание раскрывается в модальном окне) → источники («Мы не просим верить на слово.», реестр рецензируемых работ с DOI + слоган «Эмпирика надёжнее.») → Ozon-CTA → футер с дисклеймером. Секции «Гранулометрия», «Чем это не „универсальный грунт“», «Что эта земля даёт», отдельный научный блок «Почему мы вообще возимся с землёй» в пересобранном прототипе отсутствуют и переноситься MUST NOT. Контент SHALL храниться в `src/content/lab.ts`.

#### Scenario: Порядок секций
- **WHEN** отрендерена `/lab`
- **THEN** секции идут в порядке: hero → проблема → 7 рецептур → 11 компонентов → источники → Ozon-CTA, ровно один h1, навигация-glance содержит три пункта (Рецептуры · Компоненты · Источники)

#### Scenario: Рецептуры и суммы
- **WHEN** выполняется unit-тест контента lab
- **THEN** присутствуют 7 рецептур N°01–N°07, сумма процентов состава каждой равна 100

#### Scenario: Компоненты
- **WHEN** отрендерена `/lab`
- **THEN** секция компонентов содержит 11 плиток, у каждой — MaterialDot-цвет и название; список компонентов покрывает 4 функциональные группы (основа/питание, воздух, влага, дренаж/каркас)

### Requirement: Состав компонента раскрывается в доступной модалке
Клик по плитке компонента SHALL открывать модальное окно (`role="dialog"`, `aria-modal="true"`) с описанием компонента и source-note. Модалка SHALL: закрываться по Esc, по клику на фон и по кнопке закрытия; удерживать фокус внутри (focus-trap на Tab/Shift+Tab); возвращать фокус на открывшую плитку при закрытии; блокировать прокрутку фона на время показа. Источник контента модалки — скрытый контейнер данных компонентов на странице (не отдельный запрос).

#### Scenario: Открытие и закрытие с клавиатуры
- **WHEN** пользователь активирует плитку компонента, затем нажимает Esc
- **THEN** модалка открывается с описанием этого компонента, а после Esc закрывается и фокус возвращается на плитку

#### Scenario: Фокус заперт внутри модалки
- **WHEN** модалка открыта и пользователь табает до последнего фокусируемого элемента и жмёт Tab
- **THEN** фокус переходит на первый фокусируемый элемент внутри модалки, не покидая её

### Requirement: Deep-link рецептуры для QR дневника и мостов товара
`/lab#rec-{slug}` (где `{slug}` — латинское имя растения: monstera, ficus, anthurium, aglaonema, spathiphyllum, zamioculcas, epipremnum) SHALL раскрывать карточку соответствующей рецептуры и временно подсвечивать её. Переход по этому якорю используется QR-кодом из печатного дневника и мостами «Весь состав → в лаборатории» со страниц товара `/collectio/[slug]`.

#### Scenario: Якорь раскрывает рецептуру
- **WHEN** пользователь открывает `/lab#rec-ficus`
- **THEN** карточка рецептуры N°02 (фикус) раскрыта, проскроллена в область видимости и подсвечена

### Requirement: Тексты — дословно из прототипа, юр-чистота
Тексты /lab SHALL переноситься из `lab.html` дословно, без перефразирования (формулировки прошли выверку evidence-base + recepty-claims v1.1; канон `lab.md` v2.0 отстаёт и источником НЕ является). Блэклист-лексика («лечит», «очищает воздух», «польза для здоровья», «гарантирует», NASA-claims) MUST NOT встречаться. Терминология состава — «биочар» (не «уголь»), согласовано с прототипами товара.

#### Scenario: Ключевые выверенные формулировки на месте
- **WHEN** прогоняется контент-тест `/lab`
- **THEN** научный якорь проблемы (Bugbee & Frink) присутствует, блэклист-слов нет, слово «уголь» в составе не встречается

### Requirement: Дисклеймер на /lab
Футер страницы `/lab` SHALL содержать дисклеймер дословно: «Растения — не лекарство. Мы опираемся на исследования о связи контакта с природой и самочувствия, но не обещаем терапевтического эффекта.» На остальных страницах дисклеймер MUST NOT рендериться (до решения Насти «весь сайт vs /lab»).

#### Scenario: Дисклеймер локален
- **WHEN** отрендерены `/lab` и `/`
- **THEN** дисклеймер присутствует в футере /lab и отсутствует в футере главной
