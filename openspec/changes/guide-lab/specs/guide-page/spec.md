# guide-page — spec (new)

## ADDED Requirements

### Requirement: Страница гайда по прототипу guide.html
`/guide` SHALL содержать: hero («Гайд по пересадке», H1-строка «Руки в землю — голова свободна.») → тировую структуру → CJM из 5 стадий 00–04 с заголовками h2 «Подготовка», «Конверт», «Дренаж», «Грунт», «Забота» → Ozon-блок → футер-мост «← На главную». Каждая стадия SHALL включать «Что понадобится» с маркировкой ● «в боксе» / ○ «своё», шаги, подсказку (details) и критерий «Готово, когда». Тексты SHALL браться дословно из `guide.md` v3.0 и храниться в `src/content/guide.ts`.

#### Scenario: Пять стадий CJM
- **WHEN** отрендерена `/guide`
- **THEN** присутствуют ровно 5 стадий-заголовков h2 (Подготовка, Конверт, Дренаж, Грунт, Забота) в этом порядке, один h1, без пропуска уровней заголовков

#### Scenario: Маркировка инвентаря
- **WHEN** отрендерена стадия с инвентарём
- **THEN** позиции из бокса помечены ●, свои — ○

### Requirement: HowTo-разметка
Страница SHALL содержать JSON-LD Schema.org `HowTo` с 5 шагами, генерируемый из тех же данных `guide.ts`, что и видимый контент.

#### Scenario: Разметка валидна и синхронна
- **WHEN** отрендерена `/guide`
- **THEN** в HTML присутствует script `application/ld+json` типа HowTo с 5 шагами, имена шагов совпадают с заголовками стадий

### Requirement: Ноль рукописных пасхалок
Страница MUST NOT содержать RitualNote/рукописных акцентов (правило канона guide: Caveat = 0).

#### Scenario: Пасхалок нет
- **WHEN** прогоняется контент-тест `/guide`
- **THEN** RitualNote-элементов на странице нет
