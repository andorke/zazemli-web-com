# guide-lab

## Why

`/guide` и `/lab` — последние контентные заглушки MVP, при этом обе страницы обязательны к запуску: лендинг ссылается на «почему именно так → /lab» (claim «сверено с исследованиями» без /lab ведёт в никуда — зависимость запуска, решение Насты 2026-06-27), а гайд — ядро продуктовой ценности бокса. Vault дал всё: прототипы `guide.html` и `lab.html` + канон `guide.md` v3.0 (дословно синхронизирован с прототипом 2026-07-05).

## What Changes

- `/guide` из заглушки становится страницей по `guide.html` + `guide.md` v3.0: 3 тира, Тир-3 = CJM 5 стадий 00–04 (Подготовка · Конверт · Дренаж · Грунт · Забота), маркировка ● «в боксе» / ○ «своё», каждая стадия «Что понадобится → Шаги → Подсказка → Готово, когда», Schema.org HowTo, Caveat-пасхалок 0.
- `/lab` из заглушки становится страницей по `lab.html` (единственная страница, где прототип — источник и вёрстки, и текста: канон `lab.md` v2.0 отстаёт, его переписывание — pending на стороне vault): HERO → проблема (научный якорь аэрации) → 11 компонентов (аккордеон 4 группы + source-notes с DOI) → 7 рецептур N°01–07 → «что земля даёт» → «Эмпирика надёжнее» → «чем это не универсальный грунт» → список литературы → Ozon-CTA → дисклеймер в футере /lab.
- Дисклеймер «Растения — не лекарство…» появляется на `/lab` (только там — дефолт брейнсторминга, вопрос «весь сайт vs /lab» у Насти).
- Контент — `src/content/guide.ts` и `src/content/lab.ts` с типами.

## Capabilities

### New Capabilities

- `guide-page`: страница гайда пересадки — тиры, CJM-стадии, маркировка бокс/своё, HowTo-разметка.
- `lab-page`: страница лаборатории — компоненты, рецептуры, научные источники, юр-ограничения формулировок, дисклеймер.

### Modified Capabilities

- `site-shell`: требование «Заглушки без выдуманной копи» сужается до `/diary-signup` (последняя заглушка MVP).

## Impact

- `src/app/guide/page.tsx`, `src/app/lab/page.tsx` — полная замена заглушек.
- Новые компоненты `components/sections/guide/*`, `components/sections/lab/*`; переиспользование атомов (details-аккордеон, MaterialDot, RitualNote).
- `src/content/guide.ts`, `lab.ts` — новые.
- Тесты: unit контента (юр-блэклист особенно критичен для /lab), e2e обеих страниц.
- Зависит от: `ds-migration`. Архивировать после `product-pages` (оба модифицируют «Заглушки» site-shell).
- Юр-примечание: формулировки /lab — строго из прототипа (они прошли выверку recepty-claims v1.1 и evidence-base); финальный COO-sign-off — процедура перед публикацией, не блокер вёрстки.
