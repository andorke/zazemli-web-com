# guide-lab — design

## Context

`/guide` и `/lab` — заглушки. Прототипы: `guide.html` (компактный, 3 details) и `lab.html` (пересобран 2026-07-05 по ресёрч-синтезу — самая контентная и самая интерактивная страница сайта). Канон: `guide.md` v3.0 = прототип дословно; `lab.md` v2.0 отстаёт (пример одной монстеры) — для /lab источник текста прототип. Полный контекст — `docs/superpowers/specs/2026-07-05-site-redesign-design.md`.

**Пересборка /lab (важно):** прототип `lab.html` переструктурирован после первой редакции этого change. Порядок инвертирован (рецептуры — главный ответ — идут первыми, компоненты после); 11 компонентов переехали из аккордеона в фото-грид с модальным окном описания; секции «Гранулометрия», «Чем это не универсальный грунт», «Что земля даёт» и отдельный научный блок про благополучие удалены. Инвариант «≥25 нативных details» из первой редакции более неверен — заменён на проверку 7 рецептур-карточек + 11 плиток компонентов + доступной модалки.

## Goals / Non-Goals

**Goals:**
- Обе страницы соответствуют прототипам; юр-чистота /lab (формулировки прошли выверку evidence-base/recepty-claims в прототипе — не «улучшать» и не перефразировать).
- Контент типизирован (`guide.ts`, `lab.ts`); аккордеоны рецептур, source-notes и /guide — нативные `<details>`; описание компонентов /lab — доступная модалка.
- Schema.org HowTo на /guide.

**Non-Goals:**
- Переписывание канона `lab.md` (pending на стороне vault — вопрос Насте).
- Реальные иллюстрации CJM-стадий (MJ отвергнут, рисуют заново) — слоты.
- COO-sign-off формулировок — процедура публикации, вне вёрстки.

## Decisions

1. **Тексты /lab переносятся из прототипа дословно, без редактуры.** Каждая формулировка (сфагнум «смягчён», биочар «помогает сопротивляться») прошла юр-выверку; перефразирование = юр-риск. Контент-тест фиксирует блэклист и ключевые claim-строки.
2. **Порядок секций /lab — фактический порядок пересобранного прототипа** (inverted pyramid): hero («Лаборатория грунта.») → проблема («Универсальный грунт не под твоё растение.», две колонки: поток + pullbox с научным якорем аэрации Bugbee & Frink) → **7 рецептур N°01–07** («По одной земле на каждое растение.», карточки с инлайн-раскрытием + полоска долей g4 + source-notes с DOI) → **11 компонентов** («Из чего собраны эти земли.», фото-грид плиток + модальное окно описания) → источники («Мы не просим верить на слово.», реестр работ + DOI, слоган «Эмпирика надёжнее.») → Ozon-CTA → футер с дисклеймером. Удалённые в пересборке секции (Гранулометрия, «Чем это не универсальный грунт», «Что даёт», научный блок про благополучие) — не переносить.
3. **Дисклеймер — в футере только /lab** (дефолт брейнсторминга). Реализация: слот дисклеймера в SiteFooter (пропс/слот), включаемый страницей — глобальный футер не форкается.
4. **/guide: 3 тира + CJM 00–04**, каждая стадия — единый компонент-паттерн «Что понадобится (●/○) → Шаги → Подсказка (details) → Готово, когда»; HowTo-разметка JSON-LD (5 шагов) генерируется из контента.
5. **Компоненты /lab — фото-грид (11 плиток-кнопок) + модальное окно описания**, не аккордеон. Модалка обязана быть доступной: `role="dialog"`/`aria-modal`, закрытие по Esc/фону/крестику, focus-trap на Tab, возврат фокуса на плитку, scroll-lock фона. Это единственный на сайте компонент с нетривиальным клиентским JS-состоянием — реализуется как клиентский компонент; source-note компонента внутри модалки остаётся нативным `<details>`. Рецептуры при этом — карточки с инлайн-раскрытием (`<details>`), не модалка.
6. **Deep-link рецептур `/lab#rec-{латинский-slug}`** раскрывает и подсвечивает нужную карточку (2.4с). Используется QR из печатного дневника и мостами «Весь состав → в лаборатории» со страниц товара `/collectio/[slug]` — межзависимость с change `product-pages` (там мосты уже ведут на `/lab#rec-{slug}`). id рецептур на /lab обязаны совпадать со slug'ами товара.
7. **Нумерация рецептур на /lab — N°01…N°07** (как в прототипе; отличается от товарных N°001 — уже в вопросах Насте).

## Risks / Trade-offs

- [Объём контента /lab велик, риск дрейфа при переносе] → перенос по секциям с diff-сверкой текста против прототипа (скрипт-выгрузка текстовых узлов прототипа как эталон контент-теста).
- [Канон lab.md однажды перепишут — тексты разойдутся] → зафиксировано в вопросах Насте; источник для репо остаётся прототип до явного решения.
- [HowTo-разметка может разойтись с видимым контентом] → JSON-LD строится из тех же данных `guide.ts`.
- [Модалка состава — единственный нетривиальный клиентский JS, риск a11y-регрессий] → доступность фиксируется e2e/unit: открытие с клавиатуры, Esc, focus-trap, возврат фокуса; вне модалки страница остаётся статичной (mvp_no_animation).
- [id рецептур /lab и slug'и товара разъедутся — мосты и QR сломаются] → контент-тест сверяет 7 id `rec-{slug}` со списком slug'ов товара.
