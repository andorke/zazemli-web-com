# landing-redesign — design

## Context

Главная реализована по Figma 185:2 (9 секций, `components/sections/home/`). Новый эталон — `landing.html` (структура и вёрстка) + `home.md` v2.1 (тексты). DS уже мигрирована change'ом `ds-migration` (токены v1.1.0, Newsreader/Commissioner, topbar/footer). Полный контекст — `docs/superpowers/specs/2026-07-05-site-redesign-design.md`.

## Goals / Non-Goals

**Goals:**
- Главная структурно и визуально соответствует `landing.html` на 1440px, адаптив до 360px.
- Тексты канона v2.1 дословно; контент в `home.ts` типизирован.
- Пройден прогон «6 линз» перед показом Насте (вместе с остальными страницами).

**Non-Goals:**
- Страницы товара (change `product-pages`) — до них карточки галереи ведут на `#collectio`-якорь без 404.
- Реальные фото/иллюстрации — слоты `.svis`-паттерна (aspect-ratio, без CLS).
- Скролл-анимации сверх существующего reveal-паттерна (IntersectionObserver уже в репо).

## Decisions

1. **Секции по одной, старые имена сохраняются где блок жив** (hero, sku-gallery, different-soil, whats-in-box, what-soil-gives, about, teasers, ozon-cta — перевёрстка); новые: manifesto, how-it-works, photo-band; statement.tsx удаляется вместе с контентом. Рационал: git-история и тесты остаются трассируемыми, alternative «переписать папку с нуля» теряет привязку к архивным спекам.
2. **Порядок блоков — прототип, тексты — канон.** При конфликте порядка (бокс/колбы переставлены в home.md) побеждает прототип (вёрстка = его зона ответственности); расхождение — в вопросы Насте.
3. **Кликабельность карточек — да (дефолт брейнсторминга).** До `product-pages` href временно `#collectio`-локальный (не битые ссылки); после — `/collectio/[slug]`. Это единственная временная связка между changes, помечается TODO-инвариантом в тесте (карточки должны стать ссылками на slug после появления роута).
4. **Колбы — существующий механизм рендера** (PNG-колба + cavity-masked заливки, уже реализовано коммитом `4cfa5fd`), данные трио на лендинге: антуриум N°03 / фикус N°02 / замиокулькас N°06 (по прототипу; было — другой набор).
5. **Нумерация на лендинге — «N° 01»** (с пробелом, 2 цифры) как в прототипе; на страницах товара останется N°001 — вопрос единого формата у Насти, вёрстка не блокируется.
6. **Тизер «Дневник» — текст без `<a>`**: прототип не даёт ссылки, инвариант noindex/`diary вне навигации` не нарушается.

## Risks / Trade-offs

- [Тексты прототипа vs канона местами расходятся] → чек-таблица «блок → источник текста» в первой задаче; канон дословно, прототип — только для отсутствующих блоков.
- [Битые ссылки карточек до product-pages] → временный href `#collectio` + тест, который перевернётся при появлении роута.
- [Расхождение порядка блоков с каноном home.md] → вопрос Насте; выбран прототип.

## Open Questions

- Порядок «колбы ↔ что в боксе» (прототип vs home.md) — на подтверждение Насте.
- Единая нумерация N° (лендинг «N° 01» vs товар «N°001») — вопрос Насте, не блокер.
