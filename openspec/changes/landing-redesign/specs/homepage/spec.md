## ADDED Requirements

### Requirement: Одиннадцать блоков главной по прототипу landing.html
Главная SHALL состоять из одиннадцати блоков в порядке прототипа `../zazemli-vault/Айти/Сайт/prototypes/landing.html`: Hero (фото-фон, eyebrow «Бокс для пересадки растения», H1, CTA «К коллекции →» + прайс-строка) → Манифест («Пересадка — не дело из списка…», split «Для растения»/«Для тебя») → «Как это работает» (3 шага: Выбираешь растение / Пересаживаешь по гайду / Ведёшь дневник) → Галерея `#collectio` → Колбы «Разным растениям — разная земля.» → «Что в боксе» («Всё на одну пересадку.») → Атмосферный баннер (фото-слот) → «Что даёт» («Растению»/«Тебе») → «О нас» (+ подпись «— Настя, основательница ❦») → Тизеры ×3 → Ozon-CTA. Каждый блок — отдельный компонент в `components/sections/home/`.

#### Scenario: Блоки в порядке прототипа
- **WHEN** отрендерена `/`
- **THEN** одиннадцать блоков присутствуют в DOM в указанном порядке, секции Statement нет

#### Scenario: Якорь галереи
- **WHEN** открыт URL `/#collectio`
- **THEN** браузер скроллит к секции галереи (у секции id="collectio")

### Requirement: Тексты — из канона home.md, контент отделён
Тексты главной SHALL браться дословно из `../zazemli-vault/Маркетинг/Каналы/Сайт/home.md` v2.1 и храниться в `src/content/home.ts`; для блоков, отсутствующих в каноне («Как это работает», атмосферный баннер), тексты SHALL браться из прототипа `landing.html`. Компоненты секций MUST NOT содержать строковые литералы контента.

#### Scenario: Ключевые строки канона присутствуют
- **WHEN** отрендерена `/`
- **THEN** на странице есть H1 «Заземли растение. Заземли себя.», манифест «Пересадка — не дело из списка.», заголовок галереи «Семь растений — семь рецептов земли.», заголовок колб «Разным растениям — разная земля.» и заголовок бокса «Всё на одну пересадку.»

#### Scenario: Запрещённая лексика отсутствует
- **WHEN** прогоняется контент-тест по `src/content/*.ts`
- **THEN** слова из блэклиста BUILD-SPEC («премиум», «осознанность», «лечит», «уникальный», «гарантирует», «очищает воздух», «польза для здоровья») не встречаются

### Requirement: Мост доверия на /lab
Секция колб SHALL содержать ссылку «почему именно так →» на `/lab` цветом `moss-ink`.

#### Scenario: Мост ведёт в лабораторию
- **WHEN** пользователь кликает «почему именно так →»
- **THEN** открывается `/lab`

### Requirement: Тизер «Дневник» без ссылки
Блок тизеров SHALL содержать три тизера: «Лаборатория» → `/lab`, «Гайд» → `/guide`, «Дневник» — текстовый, без ссылки (вход в дневник — только по QR из бокса).

#### Scenario: Дневник не является ссылкой
- **WHEN** отрендерен блок тизеров
- **THEN** тизеры «Лаборатория» и «Гайд» — ссылки, тизер «Дневник» не содержит `<a>` и ссылок на `/diary-signup`

## MODIFIED Requirements

### Requirement: Галерея семи SKU
Галерея `#collectio` SHALL отображать ровно 7 карточек (по одной на SKU) из `content/sku.ts` — фото-слот, номер в формате «N° 01»…«N° 07», имя растения строчными — и восьмую карточку-приглашение «N° 08 — ?» («Твоего растения нет в коллекции?»). Карточки SKU SHALL быть кликабельными и вести на `/collectio/[slug]` (до появления роута — на `#collectio`). Под гридом SHALL быть CTA «Вся коллекция →». SKU-цвета на главной MUST NOT использоваться (только moss-акцент). Eyebrow секции: «Collectio Zazemli · Партия 0».

#### Scenario: Семь карточек плюс приглашение
- **WHEN** отрендерена секция галереи
- **THEN** карточек SKU ровно 7, восьмая — приглашение «N° 08 — ?», и каждая SKU-карточка кликабельна

#### Scenario: SKU-цвета не применяются
- **WHEN** выполняется ds-lint по компонентам главной
- **THEN** SKU-цветовые переменные в них не используются
