# diary-signup-form

## Why

`/diary-signup` — точка сбора email по QR из печатного дневника (партия 0 уже в печати). Брейнсторм-спека держала её заглушкой с noindex по двум причинам: «нет прототипа» и «юр-переделка формы на 2 чекбокса pending». **Обе причины сняты**: vault дал прототип `prototypes/diary-signup.html` (конверсия-first, каркас по 7-стрим ресёрчу) и финализировал юр-требования — `Маркетинг/Каналы/Сайт/diary-signup.md` v2.2 (verbatim-тексты чекбоксов) + `PDN-152FZ-SPEC.md` (согласия, фиксация, отписка). Без формы клик-путь QR → подписка не работает.

## What Changes

- `/diary-signup` из заглушки становится страницей-формой по прототипу (noindex и «вне навигации» сохраняются, вход — по QR): hero + форма высоко (above the fold на мобиле) → «Почему тебе» (3 выгоды) → «Что внутри» (таймлайн 7 писем) → эхо + повтор CTA.
- **Форма**: поле email с валидацией формата и микрокопи состояний дословно из `error-message.md` v1.0.1; **два непредзаполненных чекбокса** — CB1 (согласие на обработку ПДн, 152-ФЗ: называет оператора «ИП Минетто А. А.» + ссылка на `/privacy`), CB2 (рекламная рассылка, 38-ФЗ: явно помечен «рекламная рассылка» + отписка). Submit заблокирован, пока не отмечены **оба**. Успешный submit подменяет форму confirmation-блоком («Записали. Первое письмо — в почте через час.»).
- **Попап-резюме политики** «Коротко о данных» (по клику на ссылку в CB1) со ссылкой «Читать полную политику →» на `/privacy`.
- Event `diary_signup_submit` на успешный submit (Я.Метрика, за consent-gate каркаса).
- **Граница фронт↔инфра**: реальный сетевой submit и фиксация согласия (email + дата + IP + формулировка/версия, хранение ≥3 года — PDN-SPEC §1.3) требуют РФ-эндпоинта/БД и РФ-сервиса рассылки (Sendsay). Эта часть — **за интеграционной точкой**: до готовности РФ-бэкенда форма собирается и валидируется, но реальная отправка выключена (заглушка отправки), сбор email не деплоится. Порядок PDN-SPEC §4: политика live → уведомление РКН (Настя) → включение сбора.

## Capabilities

### New Capabilities

- `diary-signup`: страница-форма подписки на дневник — поля, валидация, два согласия (152-ФЗ + 38-ФЗ), состояния/микрокопи, confirmation, попап политики, таймлайн 7 писем, event.

### Modified Capabilities

- `site-shell`: требование «Заглушки без выдуманной копи» сужается — `/diary-signup` больше не заглушка (реализована формой). Требование «diary-signup вне навигации и индекса» сохраняется без изменений (noindex, вне шапки/футера, вход по QR).

## Impact

- `src/app/diary-signup/page.tsx` — заглушка → страница-форма (noindex сохраняется).
- `src/content/diary.ts` — hero, gift, 3 выгоды, таймлайн 7 писем, эхо, микрокопи состояний (валидация/network/5xx/maintenance/duplicate/confirmation) дословно из канонов.
- `src/components/sections/diary/*` — клиентская форма (валидация email, чекбоксы, блокировка submit, состояния, confirmation), попап-резюме политики (модалка, Esc/фон/крестик), таймлайн.
- `src/components/site/` — cookie-нотис и футер общие (ссылка на `/privacy` заводится в `privacy-page`).
- `src/lib/metrika*` — event `diary_signup_submit` за consent-gate.
- Интеграционная точка submit — обёртка POST на РФ-эндпоинт, по умолчанию выключена (флаг/`null`) до готовности бэкенда.
- Тесты: unit (2 чекбокса не предзаполнены, микрокопи состояний дословно, 7 писем таймлайна, тексты чекбоксов = `diary-signup.md` v2.2), e2e (submit заблокирован без обоих согласий; невалидный email → ошибка + focus; happy-path → confirmation; ссылка CB1 и попап ведут на `/privacy`; `noindex` сохранён, страницы нет в навигации/sitemap).
- Зависит от: `privacy-page` (ссылки на `/privacy` из CB1 и попапа), `ds-migration` (токены/шрифты — форма внутри DS). Бэкенд-обвязка (лог согласия, Sendsay, РФ-хостинг, уведомление РКН) — вне вёрстки, сторона Насты/инфры; реальный сбор включается после их готовности.
