# analytics-consent

## ADDED Requirements

### Requirement: Cookie-баннер с отклонением по умолчанию
При первом визите система SHALL показывать cookie-баннер с двумя действиями: «Принять» и «Только необходимые». Пока пользователь не дал согласие, необязательные cookie (включая Метрику) MUST NOT устанавливаться. Выбор SHALL сохраняться и баннер повторно не показывается.

#### Scenario: Первый визит без действия
- **WHEN** пользователь открыл сайт и не взаимодействовал с баннером
- **THEN** баннер видим, скрипт Метрики не загружен, cookie аналитики отсутствуют

#### Scenario: Отказ
- **WHEN** пользователь нажимает «Только необходимые»
- **THEN** баннер скрывается, выбор сохранён, Метрика не загружается и при следующих визитах

#### Scenario: Повторный визит
- **WHEN** пользователь с сохранённым выбором открывает сайт снова
- **THEN** баннер не показывается

### Requirement: Метрика загружается только после согласия
Скрипт Яндекс.Метрики SHALL подключаться только при выполнении двух условий: пользователь нажал «Принять» И задан `NEXT_PUBLIC_METRIKA_ID`. Иные системы аналитики MUST NOT подключаться.

#### Scenario: Согласие дано
- **WHEN** задан `NEXT_PUBLIC_METRIKA_ID` и пользователь нажал «Принять»
- **THEN** счётчик Метрики инициализируется

#### Scenario: ID не задан (дев-режим)
- **WHEN** `NEXT_PUBLIC_METRIKA_ID` не задан и пользователь нажал «Принять»
- **THEN** скрипт Метрики не загружается, ошибок в консоли нет

### Requirement: Каркас целей воронки
`lib/metrika.ts` SHALL экспортировать обёртку `reachGoal(goal)` для семи точек воронки (QR-заходы считаются по UTM на стороне Метрики; клики — целями). Без согласия или без счётчика вызов SHALL быть безопасным no-op. В каркасе цель вешается на клик «Купить на Ozon»; остальные цели расставляются по мере появления страниц.

#### Scenario: Цель без согласия
- **WHEN** согласие не дано и вызывается `reachGoal('ozon-click')`
- **THEN** вызов не бросает ошибок и ничего не отправляет
